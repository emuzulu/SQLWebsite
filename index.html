What is SQL injection?

SQL injection is a web security vulnerability that allows an attacker to interfere with the queries that an application makes to its database. It allows the person executing the attack to access information that they normally should not be able to access. In some cases it can allow the attacker to delete, tamper with the contents of your database or even delete the whole database altogether!

Example

Consider an application that allows users to log in with username and password. If a user "janedoe" logs in with their password "origami123" then the application would run the SQL query below. SELECT * FROM users WHERE username = 'janedoe' AND password = 'origami123' The attacker can provide an input for username like "'janedoe' OR 1=1 --" and any password text such that the application would have to run the following command SELECT * FROM users WHERE username = 'janedoe' OR 1=1 -- AND password = "random-text-here" This command would retrieve every user and password in your database and allow access for that to the attacker. Since everything in the same like after -- in treated as a comment no error would occur. The attacker could also give a username like "'janedoe'; DROP table; --" and a random password again and in some cases they might be able to delete your whole table of users. This is because the script would look like this. SELECT * FROM users WHERE username = 'janedoe'; DROP table; -- AND password = "random-text-here" If the attacker manages to find what your database table name is for your application they can run this script right here and replace table with your database table names and delete the tables as well as everything in them.

How can you prevent SQL Injections

You can prevent SQL injections using the following methods
  • Using Prepared Statements

    • Prepared statements allow the server to know what the script looks like in advance such that it knows to interpret input as as input and not as part of the script. Click here to learn how to write prepared statements.

  • Sanitizing inputs

    • Sanitizing the input is an extra layer that could help make sure that the user doesn't try to use keywords or invalid charaters in order to try to SQL inject the database. This is not 100% effective and must be used with other methods of preventing SQL Injections such as using prepared statements. However it is good practice to make sure that you sanitize inputs so as to help prevent SQL Injections.
